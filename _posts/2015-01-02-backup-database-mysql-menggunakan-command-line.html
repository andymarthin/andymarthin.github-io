Adakalanya ketika server kita mengalami masalah seperti phpmyadmin error. ini terjadi beberapa minggu yang lalu, bukan server si tapi lebih tepatnya dilaptop gw karena gw gk mengunakan XAMPP/LAMPP tapi menginstall manual semua. ketika selesai oprek laptop gw kebetulan disini gw mengunakan linux dengan distro ubuntu 14.04, saat lagi mau coding gw coba buka phpmyadmin untuk membuat database. wah tiba-tiba error phpmyadminnya gk mau kebuka, langsung aja coba buka mysql melalui terminal, syukur masih bisa dibuka. Kemudian gw coba untuk memperbaiki phpmyadmin sampe muter-muter google gk dapet juga, terpaksa gw ambil jalan terakhir untuk reinstall phpmyadmin, seteleah gw reinstall wah ternyata sama aja makin pusing lah gw, terakhir gw mencoba untuk reinstall apache,mysql,php,phpmysql. tapi gw mencoba untuk melakukan backup terlebih dahulu databasenya ( soalnya ada skripsi gw disitu ) akhirnya gw memutuskan untuk mencari cara untuk melakukan backup mysql menggunakan command line, karena gw orangnya ganteng dan gk sombong jadi gw bagiin disini aja hehehehe...

Jika kamu mau melakukan backup hanya 1 database

mysqldump database_name > database_name.sql

backup lebih dari 1 database

mysqldump --databases database_one database_two > two_databases.sql

backup semua database yang ada

mysqldump --all-databases > all_databases.sql

Â RESTORE DATABASE

kamu bisa mengembalikan database yang telah dibackup mengunakan perintah dibawah ini

mysql database_name < database_name.sql

jika kamu ingin mengembalikan database dari cara backup semua database yang ada

mysql --one-database database_name < all_databases.sql
